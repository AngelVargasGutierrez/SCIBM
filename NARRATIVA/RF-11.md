# RF-11 – Emparejar exámenes calificados con alumnos matriculados.

## a) RF-11 – Emparejar exámenes calificados con alumnos matriculados.

| Campo | Detalle |
|---|---|
| **Caso de uso** | UC-11 – Emparejar exámenes calificados con alumnos matriculados. |
| **Actores** | Docente |
| **Propósito** | Vincular un examen físico que ya fue calificado por la IA con el registro oficial de un alumno matriculado en el sistema, creando al alumno si este aún no existe en la base de datos de la sección. |
| **Tipo** | Obligatorio (X) / Opcional ( ) |
| **Requisito ID (RF)** | RF-011 |
| **Versión** | 1.0 |
| **Descripción** | A partir del nombre detectado por la IA, el sistema separa lógicamente los nombres y apellidos de acuerdo a la cantidad de palabras detectadas. Luego, inscribe automáticamente al alumno en la sección correspondiente y vincula la calificación a este nuevo registro oficial. |
| **Precondición** | El examen debe haber sido procesado y debe poseer un texto en `NombreAlumno`. |
| **Postcondición** | Se crea un nuevo registro en la tabla `AlumnosMatriculados` y se actualiza la tabla de calificaciones, visualizando el emparejamiento en la tabla principal. |

### Curso normal de eventos

| # | Usuario | Sistema |
|---|---|---|
| 1 | El docente ingresa a la vista general o resumen de notas y fija la mirada en el módulo de tabla que enumera las calificaciones escaneadas. | |
| 2 | | El sistema estructura un amplio DataTables renderizando celdas donde el color del fondo de cada renglón alterna rítmicamente entre un blanco impecable `#ffffff` y un sombreado gris levísimo `#f8f9fa`, previniendo así el cansancio visual y mejorando el seguimiento en filas. |
| 3 | | En las columnas pertenecientes a registros desvinculados o huérfanos, el sistema dibuja el texto del nombre crudo entregado por la Inteligencia Artificial de Google y adjunta un elemento de acción a la derecha. |
| 4 | | El sistema presenta un botón compacto y funcional con la instrucción visual "Matricular y Vincular". Está maquetado por defecto con las especificaciones tipo de Bootstrap: clase `btn-sm` para un tamaño minimizado, una pintura azul cian o turquesa `#0dcaf0`, vértices y esquinas suavemente curvadas en su radio de `.25rem` y la fuente tipográfica en blanco puro `#ffffff`. |
| 5 | | Además, el sistema le adhiere reglas de interacción CSS, de manera que cuando el docente sobrepone el mouse encima de este (Hover), el color original vira tenuemente hacia un matiz más sombrío, alcanzando el `#31d2f2`. |
| 6 | El docente escoge un renglón particular de un alumno que conoce, centra su mouse y pulsa firmemente el botón "Matricular y Vincular". | |
| 7 | | El sistema capta la ejecución instantánea del clic y arma, a través de Javascript, un paquete de carga útil conteniendo el número de serie único del examen (`calificacionId`), mandándolo vía internet (Petición Fetch asíncrona) al núcleo del servidor. |
| 8 | | En el ambiente del backend, el controlador específico `MatricularYVincularAlumno` detiene el proceso un instante para cruzar referencias de la sesión de seguridad y buscar exhaustivamente la existencia de dicha nota en la tabla `ExamenesAlumnos`. |
| 9 | | El sistema extrae en cadena de texto limpia (`string`) lo que la IA escribió. Procede a diseccionarla y fragmentarla metódicamente usando el carácter del "espacio en blanco" como un cincel (`Split`). |
| 10 | | El algoritmo nativo de C# del sistema comienza un recuento de palabras en el fragmento extraído. En su bloque lógico deduce si existen cuatro o más divisiones; de ser afirmativo, encapsula inteligentemente las primeras dos como `Nombres` y las subsiguientes como cadena unificada en `Apellidos`. |
| 11 | | El sistema acude al modelo Entity Framework y moldea un nuevo integrante de la base de datos `AlumnoMatriculado`, grabando sus nombres ya pulidos y su clave foránea apuntando a esa sección. |
| 12 | | El sistema cierra la brecha, enganchando e inyectando la nueva clave principal generada dentro del campo `AlumnoMatriculadoId` de la hoja de nota que estaba suelta, afianzando todo con el comando `SaveChangesAsync()`. |
| 13 | | El servidor despacha un telegrama digital de regreso hacia el buscador de internet confirmando victoria, compuesto en JSON con `success = true`. |
| 14 | | El módulo frontal de la vista reacciona instantáneamente aplicando un baño de luz sobre toda la franja correspondiente en la tabla. Modifica su background a verde `#d1e7dd` usando el efecto `transition: background-color 1s ease`, el cual va desvaneciéndose hasta volver al gris original tras un segundo. |
| 15 | | Paralelamente y sin requerir la actualización global de la página, el sistema arranca (mediante manipulación del DOM) el botón azul original "Vincular", incrustando en su lugar una insignia textual estática de color oscuro ratificando el empalme. |
| 16 | | Como rúbrica final de la tarea, el sistema manda nacer desde la cúspide derecha de la ventana gráfica una cápsula de advertencia flotante. Ésta está rellenada de verde victoria `#198754`, con letras diáfanas `#ffffff`, y ostentando un marco oscurecido que arroja sombra `box-shadow: 0 .5rem 1rem rgba(0,0,0,.15)` hacia su parte inferior, disipándose un momento después. |
