# RF-15 – Exportar reportes de calificaciones a Google Sheets.

## a) RF-15 – Exportar reportes de calificaciones a Google Sheets.

| Campo | Detalle |
|---|---|
| **Caso de uso** | UC-15 – Exportar reportes de calificaciones a Google Sheets. |
| **Actores** | Docente / Administrador |
| **Propósito** | Generar dinámicamente un archivo nativo de Google Sheets con el consolidado de calificaciones, estadísticas de rendimiento y formato institucional, guardándolo directamente en el Google Drive del docente. |
| **Tipo** | Obligatorio (X) / Opcional ( ) |
| **Requisito ID (RF)** | RF-015 |
| **Versión** | 1.0 |
| **Descripción** | El sistema recopila las calificaciones filtradas, construye una matriz tabular que incluye encabezados institucionales, y la inyecta en un archivo de Google Sheets creado mediante la API de Google, aplicando formatos visuales de forma nativa. |
| **Precondición** | El usuario debe tener su cuenta de Google Workspace vinculada, con un `accessToken` válido. |
| **Postcondición** | Se crea un nuevo documento de hoja de cálculo en el Google Drive del usuario y se le proporciona el enlace para acceder. |

---

### Curso normal de eventos

| # | Usuario | Sistema |
|---|---|---|
| 1 | El docente, deseando respaldar la información de su clase, ingresa a la vista de "Reportes", aplica los filtros visuales (Curso, Sección, etc.) y observa la tabla general de resultados. | |
| 2 | | El sistema estructura un cuadro de mandos o panel superior delineado con un fondo blanco hueso `#ffffff` y delimitado estrictamente por márgenes grises sólidas `#dee2e6`. |
| 3 | | Dentro del panel de reportes, el sistema renderiza prominentemente el botón maestro que desencadenará la acción, etiquetado textualmente como "Exportar a Google Sheets". |
| 4 | | El sistema viste a este botón con la identidad visual corporativa de Google Workspace: se pinta de fondo verde bosque `#0f9d58`, su contorno del mismo tono `#0f9d58`, letras completamente blancas y un adorno a la izquierda del texto en forma de ícono vectorial miniatura (SVG o FontAwesome) representando la cuadrícula de una hoja de cálculo. |
| 5 | | El sistema programa sobre este botón un comportamiento interactivo para cuando el usuario desliza el cursor encima (hovering), haciendo que el matiz verde resplandezca aclarándose a `#15b369`. |
| 6 | El docente desplaza el mouse sobre el botón verde recién descrito y hace un clic contundente en él. | |
| 7 | | El sistema anula inmediatamente la capacidad del usuario de tocar cualquier otro control en pantalla. Consigue esto disparando un escudo oscurecedor (overlay negro translúcido `rgba(0,0,0,0.4)`) tapando toda la vista. |
| 8 | | En el epicentro de la pantalla, el sistema hace nacer un cuadro modal perteneciente a la librería externa SweetAlert2, dibujando su lienzo interno en blanco `#ffffff` con un biselado curvo de `10px` en cada esquina. |
| 9 | | El sistema exhibe dentro del cuadro modal un gráfico circular de carga incesante (spinner), pidiendo al usuario que guarde la calma mientras sucede la magia. |
| 10 | | Simultáneamente a nivel de software oculto (Backend), el controlador del reporte invoca a la memoria del servidor para conseguir el `accessToken` y lanza una consulta masiva contra las tablas relacionales de Entity Framework. |
| 11 | | El sistema procesa los datos recogidos, determinando automáticamente el promedio matemático general del curso y peinando la tabla para deducir qué estudiante obtuvo la calificación cúspide y cuál la base. |
| 12 | | El sistema orquesta todo en un molde matriz bidimensional `List<IList<object>>`, inyectándole en el margen superior los créditos correspondientes a la Universidad, Facultad y Escuela que dicta el curso. |
| 13 | | El sistema se comunica hacia afuera disparando una petición electrónica al coloso Google (`spreadsheets.create`). Aguarda los milisegundos necesarios para ser correspondido con el código de identificación de la hoja en blanco nacida en Drive. |
| 14 | | El sistema ataca de nuevo a Google (BatchUpdate), escribiendo implacablemente las celdas, instruyéndole que pinte las cabeceras de gris opaco, tiña el texto en estilo negrita `bold`, congele la primera línea para que flote siempre y redacte en tipografía `Arial` número `10`. |
| 15 | | El sistema ordena a la nube mover el archivo terminado al sub-casillero o directorio específico que la sección ostenta en Drive, dando fin al trámite en el servidor e informando de regreso con una URL limpia y usable. |
| 16 | | La vista en la pantalla del docente se altera; el anillo giratorio del modal desaparece bruscamente, siendo reemplazado por la animación de un garabato asertivo (check verde pistacho `#a5dc86`). |
| 17 | | El sistema rotula encima del modal, en caracteres negruzcos y nítidos `#545454`, el título "Exportación Exitosa", acompañado por un falso botón azul vibrante `#0d6efd` de esquinas suavizadas `4px` y letras blancas `#ffffff` que dice: "Abrir en Google Sheets". |

### Flujo Alterno (Excepción: Autenticación de Google Inválida)

| # | Usuario | Sistema |
|---|---|---|
| 1 | El docente presiona el botón "Exportar a Google Sheets". | |
| 2 | | El sistema entra en modo de carga y contacta a la API de Google Drive utilizando el `accessToken` almacenado del usuario. |
| 3 | | Los servidores de Google rechazan la petición respondiendo con un código de error `401 Unauthorized` porque la sesión caducó o el docente revocó los permisos de la App. |
| 4 | | El sistema detiene inmediatamente el proceso de exportación y oculta el spinner giratorio de la pantalla. |
| 5 | | El sistema dibuja un modal advirtiendo: "Conexión con Google caducada. Por favor, vuelva a vincular su cuenta institucional para poder exportar." |
