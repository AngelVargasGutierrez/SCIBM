# RF-08 – Calibrar examen maestro para estampado de nota y marcas de corrección.

## a) RF-08 – Calibrar examen maestro para estampado de nota y marcas de corrección.

| Campo | Detalle |
|---|---|
| **Caso de uso** | UC-08 – Calibrar examen maestro para estampado de nota y marcas de corrección. |
| **Actores** | Docente |
| **Propósito** | Permitir al docente definir visualmente las coordenadas exactas donde se dibujarán las marcas de corrección (checks/equis) para cada pregunta y el sello con la nota final en el PDF. |
| **Tipo** | Obligatorio (X) / Opcional ( ) |
| **Requisito ID (RF)** | RF-008 |
| **Versión** | 1.0 |
| **Descripción** | El usuario utiliza una interfaz gráfica interactiva superpuesta al PDF del examen para arrastrar y soltar indicadores. El sistema captura las coordenadas (X, Y) y la página de cada indicador de pregunta, así como la zona delimitada para el sello de la nota final, guardándolas en la base de datos para automatizar el estampado en los exámenes de los alumnos. |
| **Precondición** | El solucionario debe estar registrado y confirmado. |
| **Postcondición** | Las coordenadas de calibración quedan guardadas en la base de datos. El examen queda completamente configurado y listo para recibir exámenes. |

---

### Curso normal de eventos

| # | Usuario | Sistema |
|---|---|---|
| 1 | Tras confirmar el solucionario, el docente avanza haciendo clic hacia la pestaña o vista de "Calibración Visual". | |
| 2 | | El sistema responde renderizando un lienzo de trabajo a pantalla dividida: un contenedor central ancho para visualizar el examen y un bloque lateral derecho fijo para herramientas, con un color de base muy sutil `#f8f9fa`. |
| 3 | | El sistema procesa internamente el archivo original del examen en PDF, lo divide por páginas y proyecta las imágenes rasterizadas dentro del marco central que tiene un contorno de borde gris `#cccccc` y sombra exterior de caja `box-shadow: 0 4px 6px rgba(0,0,0,0.1)`. |
| 4 | | El sistema instancía y dibuja sobrepuesto en la primera página un marco rectangular semitransparente diseñado exclusivamente para captar el lugar del sello final (Stamp). |
| 5 | | Dicho marco del sello posee un relleno azul lavado `rgba(13, 110, 253, 0.3)` bordeado de forma punteada o discontinua `dashed` con línea de `2px` de espesor en tono azul sólido `#0d6efd`. |
| 6 | | Paralelamente, el sistema renderiza un arreglo de diminutos indicadores interactivos (pines), creando uno por cada pregunta registrada en el paso anterior y agrupándolos en la esquina superior. |
| 7 | | El sistema estiliza estos pines con geometría redonda perfecta `border-radius: 50%`, los rellena con un intenso color rojo `#dc3545`, centra en ellos el número de pregunta en blanco `#ffffff` y les añade una caída de sombra para crear la ilusión óptica de que están flotando. |
| 8 | El docente mueve el mouse hacia el rectángulo azul de sello, mantiene el clic presionado sobre él, y lo arrastra hasta el recuadro que diseñó en su examen para poner la calificación final. | |
| 9 | | El motor visual del sistema detecta el movimiento del cursor mediante eventos Drag-and-Drop, actualizando en milisegundos las coordenadas CSS absolutas `top` y `left` del marco del sello. |
| 10 | El docente procede a pinchar, arrastrar y soltar de a uno los pines circulares rojos, dejándolos ubicados junto a la alternativa correcta de las opciones impresas del PDF de su examen. | |
| 11 | | El sistema registra programáticamente cada nueva posición `PosX` y `PosY` del objeto al detectar la interrupción del clic, e identifica en qué bloque contenedor de página (Página 1, Página 2, etc.) fue depositado. |
| 12 | El docente desplaza el ratón hacia la sección de herramientas lateral y oprime el botón con el título "Guardar Calibración". | |
| 13 | | El sistema identifica el accionamiento de dicho botón primario (el cual destella o reacciona y está decorado con el color azul principal corporativo `#0d6efd` y texto blanco puro). |
| 14 | | El módulo de Javascript en el navegador web recopila las decenas de variables de coordenadas (X, Y, anchos, altos y páginas) enlazándolas al ID de cada pregunta, y estructura un objeto en formato JSON. |
| 15 | | El sistema empuja asíncronamente esta información al backend por medio de una invocación de tipo AJAX dirigiéndolo a la ruta de guardado `SaveCalibracionVisual`. |
| 16 | | Del lado del servidor, el controlador extrae la lista JSON y modifica registro por registro las columnas de calibración correspondientes a cada pregunta dentro de Entity Framework, para finalmente usar la instrucción `SaveChangesAsync()`. |
| 17 | | El sistema completa el ciclo enviando un flag verdadero (`true`) de retorno hacia el cliente web. |
| 18 | | El frontend detecta la señal de victoria e irrumpe en la pantalla generando un modal centrado de SweetAlert2, el cual oscurece todo con fondo transparente y dibuja una caja blanca `#ffffff` provista de una animación circular color verde manzana `#a5dc86` formando un check de "Guardado exitosamente". |

### Flujo Alterno (Excepción: Coordenadas Fuera de Rango)

| # | Usuario | Sistema |
|---|---|---|
| 1 | El docente arrastra un pin circular o el rectángulo del sello más allá de los bordes del canvas del PDF. | |
| 2 | | El sistema detecta (mediante Javascript) que las coordenadas (X, Y) calculadas superan el tamaño máximo del contenedor o adquieren un valor negativo. |
| 3 | | El sistema corrige automáticamente el valor, limitándolo al mínimo `0` o al valor máximo del alto/ancho del documento disponible. |
| 4 | | El elemento arrastrado salta visualmente hacia el borde interno más cercano, impidiendo que el docente lo pierda de vista. |
