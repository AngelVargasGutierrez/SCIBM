# RF-17 – Clasificar el estado académico de los estudiantes por nota.

## a) RF-17 – Clasificar el estado académico de los estudiantes por nota.

| Campo | Detalle |
|---|---|
| **Caso de uso** | UC-17 – Clasificar el estado académico de los estudiantes por nota. |
| **Actores** | Sistema (Automático) / Docente |
| **Propósito** | Evaluar automáticamente el promedio final de las calificaciones de cada estudiante y asignarle una etiqueta visual de estado de rendimiento para facilitar la lectura de los reportes. |
| **Tipo** | Obligatorio (X) / Opcional ( ) |
| **Requisito ID (RF)** | RF-017 |
| **Versión** | 1.0 |
| **Descripción** | Al consultar las calificaciones, el sistema agrupa internamente las notas de cada alumno y calcula su promedio general. Este promedio pasa por una regla de negocio que etiqueta al estudiante como "Desaprobado", "Regular" o "Aprobado". La vista interpreta este texto y aplica una semaforización mediante clases CSS (colores rojo, amarillo o verde) en la tabla de datos, inyectándolo además de manera similar en la exportación Excel o Sheets. |
| **Precondición** | Debe existir al menos una calificación vinculada al alumno para poder ser procesada. |
| **Postcondición** | La interfaz muestra insignias visuales (Badges) de rendimiento correspondientes al promedio de cada alumno. |

### Curso normal de eventos

| # | Usuario | Sistema |
|---|---|---|
| 1 | El docente se adentra en el módulo de reportes haciendo clic en la sección respectiva o exige visualizar el cuadro dinámico de promedios para su aula en particular. | |
| 2 | | El engranaje del sistema atiende la llamada HTTP, desencadenando una exploración exhaustiva en la gran base de datos sobre los exámenes que hayan sido ya evaluados en dicha sección. |
| 3 | | El núcleo de software arranca un algoritmo de agrupamiento lógico (LINQ GroupBy) y amarra cada examen suelto utilizando como pegamento el texto que alberga el nombre completo y apellidos del alumno. |
| 4 | | El sistema desanda la colección resultante en una iteración bucle (loop). Por cada uno de los muchachos enlistados, el sistema suma matemáticamente cada una de las calificaciones obtenidas en las Unidades 1, 2, 3... y corta el resultado entre la cantidad total de dichas evaluaciones para dar a luz al `Promedio`. |
| 5 | | El sistema pasa este promedio crudo por una máquina de reglas jerárquicas o condición ternaria múltiple en C#, la cual actúa como un juez implacable. |
| 6 | | El sistema calibra: si la cifra no supera estrictamente la marca de `10.5`, le graba en su registro temporal la palabra tajante de `"Desaprobado"`. |
| 7 | | El sistema coteja: si la nota superó el filtro anterior pero se ahogó sin llegar a `14.0`, entonces marca a la variable interna de estado con la denominación de `"Regular"`. |
| 8 | | El sistema concluye: si el puntaje resulta ser igual a `14.0` o lo corona con una cifra superior, otorga merecidamente la presea escrita rotulándola `"Aprobado"`. |
| 9 | | El sistema empaca toda esta información amalgamada y clasificada, enviándola codificada en formato textual JSON, volando por la red hasta aterrizar en el navegador Chrome o Edge del catedrático. |
| 10 | | La máscara gráfica, controlada por la herramienta DataTables de Javascript, intercepta el JSON y da inicio al ciclo de renderizado, pintando celda por celda la gran matriz en pantalla. |
| 11 | | Al momento en que el lápiz digital del sistema debe trazar la columna "Estado" o "Rendimiento", el código detiene la inyección de texto simple y analiza el vocablo. Con base a este, decide teñir la celda invocando bloques constructivos y estilos CSS puros (semaforización visual). |
| 12 | | Si se detecta **"Desaprobado"**: El sistema esculpe un rectángulo de advertencia dura (Badge) bañándolo de rojo carmesí `#dc3545`, de aristas contorneadas `border-radius: 0.25rem`, incrustando el texto en un inmaculado blanco `#ffffff` para resaltar la urgencia, rematando con cuerpo de fuente ensanchado (`font-weight: 600`). |
| 13 | | Si surge la palabra **"Regular"**: El sistema levanta el mismo bloque rectangular pero inundándolo de un color amarillo mostaza `#ffc107`, que denota cautela y alerta temprana, forzando a que la tinta de las letras mute a negro grafito `#212529` evitando así que se pierda la lectura ante el fondo claro. |
| 14 | | Si brota el término **"Aprobado"**: El sistema consolida la figura coloreándola en tonalidad verde pacífico `#198754`, reafirmando el éxito con tipografía blanquecina `#ffffff` y el mismo trazo robusto, transmitiendo seguridad. |
| 15 | El docente contempla frente a él la inmensa cuadrícula enteramente dibujada y coloreada, escudriñando con presteza (guiado instintivamente por la luminosa paleta del semáforo) sobre a qué alumnos debe extenderles la mano de inmediato. | |

### Flujo Alterno (Excepción: Alumno Sin Calificaciones)

| # | Usuario | Sistema |
|---|---|---|
| 1 | El docente ingresa a la vista de reportes de una sección nueva, donde los alumnos están formalmente matriculados pero aún no han rendido ninguna prueba. | |
| 2 | | El sistema realiza la consulta habitual cruzando datos con las tablas de calificaciones. |
| 3 | | El algoritmo de agrupamiento detecta que el arreglo de notas para ese estudiante se encuentra completamente vacío. |
| 4 | | El sistema, previniendo un error por división entre cero, asigna automáticamente un valor nulo o cero al promedio. |
| 5 | | En la máquina de estados lógicos, el sistema obvia las etiquetas habituales y le asigna el estado textual "Sin Datos". |
| 6 | | La tabla (DataTables) reacciona a este texto dibujando una celda con una insignia (Badge) gris neutro `#6c757d` que indica "No Evaluado", descartando la semaforización. |
