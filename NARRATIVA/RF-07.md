# RF-07 – Registrar el solucionario de examen maestro.

## a) RF-07 – Registrar el solucionario de examen maestro.

| Campo | Detalle |
|---|---|
| **Caso de uso** | UC-07 – Registrar el solucionario de examen maestro. |
| **Actores** | Docente |
| **Propósito** | Permitir al docente revisar, editar y confirmar definitivamente las preguntas y respuestas correctas (solucionario) extraídas del examen. |
| **Tipo** | Obligatorio (X) / Opcional ( ) |
| **Requisito ID (RF)** | RF-007 |
| **Versión** | 1.0 |
| **Descripción** | El docente revisa una tabla generada con los datos de las preguntas. Puede modificar los enunciados, respuestas y puntajes en línea. Al presionar confirmar, el sistema captura todos los datos corregidos, verifica las eliminaciones y guarda todo de forma definitiva en la base de datos, preparando el examen para su calibración visual. |
| **Precondición** | Las preguntas extraídas deben estar pre-cargadas en el sistema (RF-06). |
| **Postcondición** | El solucionario queda confirmado y guardado permanentemente en la base de datos. El sistema redirige al usuario a la fase de Calibración Visual del examen. |

---

### Curso normal de eventos

| # | Usuario | Sistema |
|---|---|---|
| 1 | El docente es redirigido a la pantalla de revisión de solucionario tras haber superado con éxito el análisis automático de la IA. | |
| 2 | | El sistema renderiza la nueva interfaz gráfica sobre un color de fondo gris tenue `#f4f6f9`, exhibiendo al centro un panel contenedor blanco `#ffffff` estilizado con sombra volumétrica `box-shadow: 0 4px 6px rgba(0,0,0,0.1)`. |
| 3 | | Dentro del panel, el sistema despliega dinámicamente una tabla de datos (DataTables). La cabecera superior (thead) de esta tabla cuenta con un fondo oscuro `#343a40` y tipografía blanca `#ffffff`. |
| 4 | | El sistema genera el cuerpo de la tabla pintando las filas alternativamente entre un color blanco `#ffffff` y un gris ultraclaro `#f8f9fa`, separados por líneas divisorias de tono `#dee2e6`. |
| 5 | | En la zona superior derecha del panel, el sistema sitúa una tarjeta o insignia (Badge) estadística que suma en tiempo real el total de puntos de las preguntas detectadas. |
| 6 | | El sistema ejecuta una validación lógica: si la sumatoria de puntaje concuerda con lo esperado (ej. 20 puntos), el Badge se pinta de color fondo verde `#198754`. De lo contrario, se colorea de rojo intenso `#dc3545` con texto blanco. |
| 7 | El docente hace clic primario en la celda de un campo de texto con la intención de enmendar un enunciado o corregir el puntaje asignado. | |
| 8 | | El sistema detecta el foco de interacción del usuario (evento focus), alterando instantáneamente el contorno del input: cambia el gris por defecto por un borde azul brillante `#86b7fe` y le irradia una sombra exterior `box-shadow: 0 0 0 0.25rem rgba(13,110,253,0.25)`. |
| 9 | El docente remueve una fila de la tabla que contiene información errónea haciendo clic sobre el botón de papelera correspondiente. | |
| 10 | | El sistema captura la acción sobre el botón (el cual posee fondo rojo `#dc3545` e ícono blanco de basurero) y mediante manipulación del DOM, desaparece la fila instantáneamente frente al usuario. |
| 11 | El docente, una vez seguro de los datos, presiona el botón de acción principal que indica "Confirmar Solucionario". | |
| 12 | | El sistema procesa el clic sobre este botón, diseñado con fondo color verde `#198754`, texto blanco y acompañado de un pequeño ícono de aspa de validación. |
| 13 | | Mediante un script frontend en Javascript, el sistema barre toda la tabla activa, extrayendo fila por fila cada uno de los valores ingresados. |
| 14 | | El sistema estructura la información y envía un paquete por el método asíncrono POST al controlador backend `ConfirmarSolucionario`. |
| 15 | | A nivel de servidor, el sistema recupera la lista persistida original y la compara con el nuevo arreglo, aplicando comandos de borrado (Remove) a aquellas entidades de base de datos que ya no figuran en la solicitud. |
| 16 | | El sistema itera las preguntas remanentes y sobreescribe sus atributos (Enunciado, Respuesta, Puntaje), aplicando una normalización lógica que sustituye comas por puntos en los valores decimales. |
| 17 | | El sistema finaliza aplicando un `SaveChangesAsync()`, grabando permanentemente las modificaciones en Entity Framework. |
| 18 | | El sistema refresca la ruta de la aplicación dirigiendo al docente hacia la interfaz de "Calibración" y dibuja una alerta emergente flotante en esquina (Toast) con fondo verde `#51a351` informando el éxito del procedimiento. |
