# RF-06 – Analizar plantilla de examen.

## a) RF-06 – Analizar plantilla de examen.

| Campo | Detalle |
|---|---|
| **Caso de uso** | UC-06 – Analizar plantilla de examen. |
| **Actores** | Docente |
| **Propósito** | Permitir al docente subir un examen resuelto (solucionario en PDF) para que la Inteligencia Artificial extraiga automáticamente las preguntas y respuestas correctas. |
| **Tipo** | Obligatorio (X) / Opcional ( ) |
| **Requisito ID (RF)** | RF-006 |
| **Versión** | 1.0 |
| **Descripción** | El sistema recibe un archivo PDF del solucionario, lo guarda temporalmente, se comunica con Gemini Vision para analizar la estructura, y extrae un listado de preguntas con su tipo, enunciado, puntaje y respuesta esperada, preparando los datos para su posterior revisión en la interfaz. |
| **Precondición** | El usuario debe estar autenticado. Debe existir un Examen creado en el sistema. El usuario debe tener un archivo PDF válido que represente el solucionario maestro. |
| **Postcondición** | Las preguntas extraídas se guardan temporalmente en la base de datos vinculadas al examen actual y el sistema notifica al usuario para que proceda a la revisión visual. |

---

### Curso normal de eventos

| # | Usuario | Sistema |
|---|---|---|
| 1 | El docente ingresa a la vista principal del examen y selecciona la opción para subir un solucionario (plantilla resuelta). | |
| 2 | | El sistema carga la interfaz con un fondo general gris claro `#f4f6f9`. Muestra un panel contenedor con fondo blanco `#ffffff` y una ligera sombra `box-shadow: 0 0 10px rgba(0,0,0,0.1)`. |
| 3 | | El sistema presenta un área de carga de archivos (Dropzone) interactiva con un borde punteado azul `#0d6efd`. |
| 4 | El docente hace clic en el área de carga y selecciona un archivo PDF desde su equipo. | |
| 5 | | El sistema lee el archivo cargado y muestra el nombre y tamaño del PDF en la pantalla. |
| 6 | El docente hace clic en el botón "Analizar con IA". | |
| 7 | | El sistema detecta el clic en el botón, el cual se destaca con un fondo azul primary `#0d6efd`, texto blanco `#ffffff` y bordes redondeados `4px`. |
| 8 | | El sistema bloquea toda interacción en la pantalla oscureciendo el fondo general con un overlay negro translúcido `rgba(0,0,0,0.4)` y despliega de manera automática un modal SweetAlert2 central. |
| 9 | | Este modal de carga emerge con un fondo blanco puro `#ffffff`, bordes redondeados de `8px`, mostrando un ícono de spinner animado girando y el texto "Procesando...". |
| 10 | | El sistema en el backend (`ExamenController`) recibe el archivo a través de la solicitud POST, verifica su extensión para confirmar que sea un PDF y lo almacena en el directorio local `App_Data/Examenes/Temp`. |
| 11 | | El sistema establece comunicación segura con el servicio de IA (`GeminiApiService`), transfiriéndole el archivo PDF y la instrucción base (Prompt) para extraer las preguntas. |
| 12 | | El sistema recibe la respuesta de la Inteligencia Artificial en formato de texto JSON, la decodifica y comienza a iterar para mapear cada pregunta (incluyendo enunciado, puntaje, tipo y respuesta correcta). |
| 13 | | El sistema almacena estas preguntas mapeadas en la base de datos relacional de manera temporal vinculándolas a la entidad del examen actual. |
| 14 | | El sistema ordena el cierre automático del modal de spinner y lanza de inmediato una alerta flotante lateral (tipo Toast) en la esquina superior derecha del monitor. |
| 15 | | La alerta Toast se presenta con un fondo color verde muy claro `#d1fae5`, con un borde izquierdo ancho y grueso en color verde oscuro `#198754` y con el texto de confirmación renderizado en tonalidad verde `#0f5132`. |
| 16 | El docente lee visualmente el mensaje de éxito. | |
| 17 | | El sistema redirige automáticamente la ventana del navegador web hacia la vista detallada de confirmación de preguntas. |


---

### Flujo Alterno (Excepción: Archivo inválido o sin preguntas)

| # | Usuario | Sistema |
|---|---|---|
| 1 | El docente sube un archivo que no es PDF o es un PDF escaneado sin texto legible (solo imágenes pixeladas). | |
| 2 | | El sistema envía el archivo a la IA para su análisis. |
| 3 | | La IA responde con un arreglo vacío al no encontrar estructura de examen ni texto identificable. |
| 4 | | El sistema oculta el modal de carga y despliega una alerta SweetAlert roja de "Error". |
| 5 | | El sistema indica al usuario: "No se encontraron preguntas legibles. Asegúrese de que el PDF contenga texto." |
| 6 | El docente debe volver a intentarlo subiendo un archivo PDF válido. | |
